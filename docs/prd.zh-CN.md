# memduck 产品需求文档

## 一句话

把链接、长帖、图文、截图、视频等外部碎片内容，自动消化为可问答、可回顾、会随着使用不断变深的私人记忆。

## 产品定义

memduck 是一个开源、自部署、尽量开箱即用的个人信息消化引擎。

它不是传统笔记软件，不是普通收藏夹，也不是单纯的稍后读工具。核心目标不是“帮用户存起来”，而是：

- 先理解
- 再压缩
- 保留真正有价值的部分
- 支持未来通过问答和回顾重新调用

## 目标用户

- 重度信息摄入者
- 创作者、内容策划、写作者
- 研究型个人用户
- 开发者与开源用户

第一阶段不优先服务团队知识库、企业协作和纯轻度收藏场景。

## 产品原则

- 先消化，再归档
- 原始来源必须可追溯
- 默认轻量，逐步加深
- 让结果比原文更值得再次打开
- 记忆会随着使用而变化
- 尽量减少客户端负担

## MVP 形态

- Web 主界面
- 浏览器插件
- Telegram 机器人

第一阶段不做原生 App，不做多用户工作区，不把主动定时推送作为主链路。

## MVP 输入

- `url`
- `text`
- `image`

所有入口统一进入同一条记忆流水线：

1. 输入采集
2. 内容规范化
3. 保存原始来源
4. 消化与摘要
5. 生成记忆卡
6. 主题关联
7. 问答与回顾

## MVP 核心能力

- 链接摘要
- 文本摘要
- 截图/图片轻量理解
- 重点提取
- 基础主题关联
- 记忆卡生成
- 基于个人内容的问答
- 基础回顾
- 行为信号驱动的轻量加权

## 成功标准

- 几分钟内跑通“输入一条内容并拿到第一张记忆卡”
- 用户能明确感受到 memduck 不是普通收藏夹
- 用户可以回到原始来源
- 问答和回顾都体现“基于个人已存内容”
